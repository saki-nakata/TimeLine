# 投稿機能定義書

**バージョン:** 1.0
**作成日:** 2026-05-17
**作成者:** Nakata Saki

---

## 1. 機能概要

ログイン済みユーザーがテキストおよび任意の画像を投稿できる機能。
投稿はタイムライン（S-03）および投稿詳細画面（S-04）に表示される。
自分の投稿は編集・削除可能。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|---------|
| US-POST-01 | ログイン済みユーザーとして、テキストを入力して投稿ボタンを押すことで、新しい投稿を作成したい |
| US-POST-02 | ログイン済みユーザーとして、画像を添付して投稿することで、ビジュアルを含む投稿を作成したい |
| US-POST-03 | ログイン済みユーザーとして、投稿カードをクリックすることで、投稿の詳細（全文・コメント）を確認したい |
| US-POST-04 | ログイン済みユーザーとして、自分の投稿の「…」メニューから「編集」を選び、投稿内容をモーダルで修正したい |
| US-POST-05 | ログイン済みユーザーとして、自分の投稿の「…」メニューから「削除」を選び、投稿を削除したい |

---

## 3. 機能詳細

### 3.1 投稿作成

| 項目 | 内容 |
|------|------|
| 入力フォーム設置場所 | タイムライン（S-03）の上部 |
| テキスト入力 | テキストエリア。最大 280 文字 |
| 文字数カウンター | 入力中にリアルタイムで「現在の文字数 / 280」を表示 |
| 画像添付 | 「📎 画像を追加」ボタンでファイル選択。1 枚のみ添付可能 |
| 投稿ボタン | テキストが 1 文字以上入力されている場合にのみ活性化 |
| 投稿後の動作 | タイムラインの先頭に作成した投稿を即時表示（楽観的更新 or 再取得） |

### 3.2 画像アップロード

| 項目 | 内容 |
|------|------|
| 対応フォーマット | JPEG / PNG / GIF |
| 最大ファイルサイズ | 4MB |
| アップロード先 | AWS S3 |
| 保存形式 | S3 上のオブジェクト URL を `posts.image_url` に保存 |
| プレビュー | 画像選択後、フォーム内にサムネイルプレビューを表示 |

### 3.3 投稿詳細表示

| 項目 | 内容 |
|------|------|
| 遷移方法 | 投稿カードのクリック、または「コメント」ボタンのクリック |
| 表示内容 | 投稿全文・画像・投稿日時・いいね数・コメント数・コメント一覧 |

### 3.4 投稿編集

| 項目 | 内容 |
|------|------|
| 表示条件 | 編集メニューは自分の投稿の「…」ボタンにのみ表示する |
| 編集 UI | モーダルで投稿作成フォームと同じ UI を表示し、既存のテキスト・画像をプリセットする |
| テキスト | 最大 280 文字。編集して「保存」ボタンで更新 |
| 画像 | 既存画像の変更・削除が可能 |
| 保存後の動作 | タイムライン・投稿詳細の表示をリアルタイム更新 |

### 3.5 投稿削除

| 項目 | 内容 |
|------|------|
| 表示条件 | 削除メニューは自分の投稿の「…」ボタンにのみ表示する |
| 削除確認 | 確認ダイアログを表示してから削除する |
| S3 画像の扱い | 投稿削除時に S3 の画像ファイルも削除する |
| 削除後の動作 | タイムラインから当該投稿を除去 |

---

## 4. 使用 API エンドポイント

| Method | エンドポイント | 説明 |
|--------|--------------|------|
| POST | `/api/posts` | 投稿作成（multipart/form-data） |
| GET | `/api/posts/{id}` | 投稿詳細取得 |
| PUT | `/api/posts/{id}` | 投稿編集（自分の投稿のみ） |
| DELETE | `/api/posts/{id}` | 投稿削除（自分の投稿のみ） |
| POST | `/api/images/upload` | 画像アップロード（S3） |

---

## 5. バリデーション仕様

### フロントエンド（入力時・送信前）

| 対象 | ルール | エラーメッセージ |
|------|--------|----------------|
| テキスト | 必須（1 文字以上） | 「投稿内容を入力してください」 |
| テキスト | 最大 280 文字 | 「280 文字以内で入力してください（現在 xxx 文字）」 |
| 画像 | JPEG / PNG / GIF のみ | 「JPEG・PNG・GIF 形式のファイルを選択してください」 |
| 画像 | 4MB 以下 | 「ファイルサイズは 4MB 以下にしてください」 |

### バックエンド（API 受信時）

| 対象 | ルール | HTTP ステータス |
|------|--------|---------------|
| テキスト | 必須 | 400 Bad Request |
| テキスト | 最大 280 文字 | 400 Bad Request |
| 画像 MIME タイプ | JPEG / PNG / GIF | 400 Bad Request |
| 画像サイズ | 4MB 以下 | 400 Bad Request |
| 認証 | JWT が有効であること | 401 Unauthorized |

---

## 6. 制約・前提条件

| 分類 | 内容 |
|------|------|
| 認証 | 投稿作成・編集・削除はログイン済みユーザーのみ可能 |
| 編集権限 | 自分の投稿のみ編集・削除可能（他ユーザーの投稿は不可） |
| 画像枚数 | 1 投稿につき画像は 1 枚のみ |

---

## 7. 対象外仕様

| 対象外機能 | 理由 |
|-----------|------|
| 動画の添付 | スコープ外 |
| 複数画像の添付 | スコープ外 |
| 下書き保存 | スコープ外 |
