# タイムライン機能定義書

**バージョン:** 1.1
**作成日:** 2026-05-17
**更新日:** 2026-05-24
**作成者:** Nakata Saki

---

## 1. 機能概要

ホーム画面（S-03）に表示される投稿一覧。「全て」タブと「フォロー中」タブを切り替えることで、全ユーザーの投稿またはフォロー中ユーザーの投稿のみを新着順に閲覧できる。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|---------|
| US-TL-01 | ログイン済みユーザーとして、ホーム画面を開いたとき、全ユーザーの最新投稿を確認したい |
| US-TL-02 | ログイン済みユーザーとして、「フォロー中」タブに切り替えることで、フォロー中のユーザーの投稿だけを確認したい |
| US-TL-03 | ログイン済みユーザーとして、古い投稿を読み込むために「もっと読み込む」ボタンを押したい |

---

## 3. 機能詳細

### 3.1 タブ切り替え

| 項目 | 内容 |
|------|------|
| タブ種類 | 「全て」「フォロー中」の 2 種類 |
| デフォルト | 「全て」を選択状態で表示 |
| 切り替え時 | タブをクリックするとそのタブの投稿一覧を取得・再描画する |

### 3.2 投稿一覧の表示内容

各投稿カードに以下を表示する。

| 表示項目 | 説明 |
|---------|------|
| @username | 投稿者のユーザー名。クリックでプロフィール画面（S-05）へ遷移 |
| 投稿日時 | 相対表示（例: 3 分前）。マウスオーバーで絶対日時を表示 |
| テキスト | 投稿本文（最大 280 文字） |
| いいね数 | 現在のいいね数 |
| いいねボタン | 自分がいいね済みの場合はアクティブ（色付き）表示 |
| コメント数 | 現在のコメント数 |
| コメントボタン | クリックでコメントモーダルを開く |

### 3.3 取得順序

- 新着順（`created_at DESC`）で取得する

### 3.4 ページネーション

| 項目 | 内容 |
|------|------|
| 1 回の取得件数 | 20 件（`limit` パラメータ、デフォルト 20） |
| 方式 | カーソルベース（`cursor` に最後の投稿 ID を指定して続きを取得） |
| 追加取得 | 現在表示中のリストの末尾に追記する（ページ遷移なし） |

---

## 4. 使用 API エンドポイント

| Method | エンドポイント | パラメータ | 説明 |
|--------|--------------|-----------|------|
| GET | `/api/posts` | `type=all\|following`、`cursor`（最後の投稿 ID）、`limit`（デフォルト 20） | タイムライン取得 |

---

## 5. バリデーション仕様

タイムライン表示自体に入力バリデーションはない。

---

## 6. 制約・前提条件

| 分類 | 内容 |
|------|------|
| 認証 | ログイン済みユーザーのみ閲覧可能 |
| フォロー中タブ | フォロー中のユーザーが 0 人の場合は「投稿がありません」を表示 |
| 「全て」タブ | 自分の投稿も含む全ユーザーの投稿を表示する |

---

## 7. 対象外仕様

| 対象外機能 | 理由 |
|-----------|------|
| インプレッション数（閲覧数）の表示 | スコープ外（差別化ポイント） |
| リポスト / リツイート | スコープ外 |
| 投稿のリアルタイム更新（WebSocket 等） | スコープ外 |
| アルゴリズムによるおすすめ表示 | スコープ外 |
