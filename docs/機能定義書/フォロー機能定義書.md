# フォロー機能定義書

**バージョン:** 1.0
**作成日:** 2026-05-17
**作成者:** Nakata Saki

---

## 1. 機能概要

ログイン済みユーザーが他のユーザーをフォロー・アンフォローできる機能。
フォローしているユーザーの投稿はタイムラインの「フォロー中」タブに表示される。
フォロワー数・フォロー中数はプロフィール画面（S-05）に表示する。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|---------|
| US-FLW-01 | ログイン済みユーザーとして、他のユーザーのプロフィール画面で「フォローする」ボタンを押し、そのユーザーをフォローしたい |
| US-FLW-02 | ログイン済みユーザーとして、フォロー中のユーザーのプロフィール画面で「フォロー中」ボタンを押し、アンフォローしたい |
| US-FLW-03 | ログイン済みユーザーとして、タイムラインの「フォロー中」タブで、フォロー中のユーザーの投稿だけを確認したい |
| US-FLW-04 | ログイン済みユーザーとして、プロフィール画面でそのユーザーのフォロワー数・フォロー中数を確認したい |
| US-FLW-05 | ログイン済みユーザーとして、ユーザー検索結果画面のユーザーカードから直接フォロー操作をしたい |

---

## 3. 機能詳細

### 3.1 フォロー操作

| 項目 | 内容 |
|------|------|
| 操作場所 | プロフィール画面（S-05）・ユーザー検索結果画面（S-06） |
| フォローボタン | 「フォローする」クリックでフォロー |
| アンフォローボタン | 「フォロー中」クリックでアンフォロー確認後に実行 |
| ボタン表示条件 | 自分自身のプロフィールには表示しない |

### 3.2 ボタン表示仕様

| 状態 | 表示 |
|------|------|
| 未フォロー | 「フォローする」（アウトライン） |
| フォロー済み | 「フォロー中」（塗りつぶし）。マウスオーバーで「アンフォロー」に変更 |

### 3.3 フォロワー / フォロー中数の表示

| 項目 | 内容 |
|------|------|
| 表示場所 | プロフィール画面（S-05）・ユーザー検索結果画面（S-06） |
| データ源 | `users.follower_count`、`users.following_count`（非正規化カウンタ） |
| 更新タイミング | フォロー/アンフォロー操作後にリアルタイム反映 |

### 3.4 フォロー中ユーザー一覧

| 項目 | 内容 |
|------|------|
| 表示場所 | `/following` ページ（左サイドバー「フォロー中」から遷移） |
| 表示内容 | アバター / 表示名 / @username / bio / フォロー解除ボタン |
| データ取得 | `GET /api/users/{id}/following` |
| フォロー解除 | 一覧上でアンフォロー操作が可能。一覧の状態をリアルタイム反映 |
| 空状態 | フォロー中ユーザーがいない場合はメッセージを表示 |

### 3.5 フォロー中タイムラインへの反映（ホームタブ）

| 項目 | 内容 |
|------|------|
| フォロー後 | そのユーザーの投稿が「フォロー中」タブのタイムラインに表示される |
| アンフォロー後 | そのユーザーの投稿が「フォロー中」タブから除外される |

### 3.6 制限

| 項目 | 内容 |
|------|------|
| 自己フォロー | 自分自身をフォローすることはできない（DB 側で CHECK 制約） |
| 重複フォロー | 同じユーザーを重複してフォローすることはできない（DB 側で UNIQUE 制約） |

---

## 4. 使用 API エンドポイント

| Method | エンドポイント | 説明 |
|--------|--------------|------|
| POST | `/api/users/{id}/follows` | フォロー |
| DELETE | `/api/users/{id}/follows` | アンフォロー |
| GET | `/api/users/{id}` | プロフィール取得（フォロー状態・フォロワー数を含む） |
| GET | `/api/users/{id}/following` | フォロー中ユーザー一覧取得 |

---

## 5. バリデーション仕様

| 対象 | ルール | HTTP ステータス |
|------|--------|---------------|
| 認証 | JWT が有効であること | 401 Unauthorized |
| 自己フォロー | フォロー対象が自分自身でないこと | 400 Bad Request |
| 重複フォロー | すでにフォロー済みの場合は 200 を返す（エラーにしない） | 200 OK |
| 存在確認 | フォロー対象のユーザーが存在すること | 404 Not Found |

---

## 6. 制約・前提条件

| 分類 | 内容 |
|------|------|
| 認証 | フォロー/アンフォローはログイン済みユーザーのみ可能 |
| 自己フォロー禁止 | `follows.follower_id <> follows.following_id` の CHECK 制約で強制 |
| カウンタ更新 | `users.follower_count`、`users.following_count` はサービス層で `+1/-1` 更新する |

---

## 7. 対象外仕様

| 対象外機能 | 理由 |
|-----------|------|
| フォロワー一覧画面 | スコープ外 |
| フォロー申請・承認機能（非公開アカウント） | スコープ外 |
| フォロー通知 | スコープ外 |
