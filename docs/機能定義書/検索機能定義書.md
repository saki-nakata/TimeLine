# 検索機能定義書

**バージョン:** 1.0
**作成日:** 2026-06-25
**作成者:** Nakata Saki

---

## 1. 機能概要

ログイン済みユーザーが、ナビゲーションの検索アイコンから SearchOverlay を開き、ユーザー・投稿・ハッシュタグを横断検索できる機能。専用ページへの遷移は行わない。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|---------|
| US-SRCH-01 | ログイン済みユーザーとして、検索バーにキーワードを入力してユーザーと投稿を同時に検索したい |
| US-SRCH-02 | ログイン済みユーザーとして、検索候補のユーザーをクリックしてプロフィール画面に遷移したい |
| US-SRCH-03 | ログイン済みユーザーとして、検索候補の投稿をクリックして投稿詳細画面に遷移したい |
| US-SRCH-04 | ログイン済みユーザーとして、`#タグ` を入力してそのハッシュタグが付いた投稿を検索したい |

---

## 3. 機能詳細

### 3.1 検索操作

| 項目 | 内容 |
|------|------|
| 検索 UI | SearchOverlay（ナビゲーションの検索アイコンをクリック/タップするとトグル表示） |
| 検索方法 | 入力から 300ms debounce 後に自動検索。結果はドロップダウンで表示 |
| `#` なし入力 | ユーザー検索（username / display_name 部分一致） + 投稿本文検索を同時実行 |
| `#` あり入力 | ハッシュタグ検索（投稿のみ。ユーザー検索はスキップ） |

### 3.2 検索結果の表示

ドロップダウンを「ユーザー」「投稿」のセクションに分けて表示する。

**ユーザーセクション（`#` なし時のみ表示）**

| 表示項目 | 内容 |
|---------|------|
| アバター | アバター画像（未設定時はイニシャル表示） |
| 表示名 | `display_name`（未設定時は `username`） |
| @username | ユーザー名 |
| クリック動作 | プロフィール画面（`/profile/:userId`）へ遷移 |

**投稿セクション**

| 表示項目 | 内容 |
|---------|------|
| アバター | 投稿者のアバター画像 |
| 表示名 / @username | 投稿者情報 |
| 投稿本文 | 最大 2 行で省略表示 |
| クリック動作 | 投稿詳細画面（`/posts/:id`）へ遷移 |

### 3.3 検索結果が 0 件の場合

「見つかりませんでした」のメッセージを表示する。

### 3.4 検索ワードが空の場合

ドロップダウンを非表示にし、API 呼び出しは行わない。

---

## 4. 使用 API エンドポイント

| Method | エンドポイント | パラメータ | 説明 |
|--------|--------------|-----------|------|
| GET | `/api/users/search` | `q=xxx` | ユーザー名・表示名で部分一致検索（`#` なし時） |
| GET | `/api/posts/search` | `q=xxx`, `cursor`, `limit` | 投稿本文キーワード検索（`#` なし時） |
| GET | `/api/posts/hashtag/{tag}` | `cursor`, `limit` | ハッシュタグ別投稿検索（`#` あり時） |

---

## 5. バリデーション仕様

### フロントエンド

| 対象 | ルール | 動作 |
|------|--------|------|
| 検索ワード | 空欄の場合は API 呼び出しを行わない | ドロップダウン非表示 |
| `#` のみ入力 | `#` 以降が空の場合は API 呼び出しを行わない | — |

### バックエンド

| 対象 | ルール | HTTP ステータス |
|------|--------|---------------|
| 認証 | JWT が有効であること | 401 Unauthorized |
| `/api/posts/search` の `q` | 必須・空文字不可 | 400 Bad Request |

---

## 6. 制約・前提条件

| 分類 | 内容 |
|------|------|
| 認証 | ログイン済みユーザーのみ検索可能 |
| ユーザー検索 | username と display_name の部分一致（`ILIKE`、大文字小文字を区別しない）、最大 20 件 |
| 投稿本文検索 | content の部分一致（`ILIKE`）、最大 5 件（ドロップダウン表示） |
| ハッシュタグ検索 | `post_hashtags` テーブルを JOIN、完全一致（小文字正規化済み）、最大 5 件 |
