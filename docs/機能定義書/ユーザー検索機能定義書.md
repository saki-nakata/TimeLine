# ユーザー検索機能定義書

**バージョン:** 1.0
**作成日:** 2026-05-17
**作成者:** Nakata Saki

---

## 1. 機能概要

ログイン済みユーザーがユーザー名（@username）で他のユーザーを検索できる機能。
検索結果画面（S-06）に一致するユーザーを一覧表示し、プロフィールへの遷移やフォロー操作を行うことができる。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|---------|
| US-SRCH-01 | ログイン済みユーザーとして、ヘッダーの「ユーザー検索」ボタンから検索画面に移動し、ユーザー名を入力して検索したい |
| US-SRCH-02 | ログイン済みユーザーとして、検索結果に表示されたユーザー名をクリックして、そのユーザーのプロフィール画面に遷移したい |
| US-SRCH-03 | ログイン済みユーザーとして、検索結果画面から直接フォローボタンを押して、ユーザーをフォローしたい |

---

## 3. 機能詳細

### 3.1 検索操作

| 項目 | 内容 |
|------|------|
| 検索画面への遷移 | ヘッダーの「ユーザー検索」アイコン/ボタンをクリック |
| 検索キー | ユーザー名（`users.username`）での部分一致（前方一致 or 全体部分一致） |
| 検索方法 | 入力フォームにユーザー名を入力して「検索」ボタンをクリック |
| URL | `/search?q=xxx` |

### 3.2 検索結果の表示

| 表示項目 | 内容 |
|---------|------|
| アバター | アバター画像（未設定時はデフォルト画像） |
| @username | ユーザー名。クリックでプロフィール画面（S-05）へ遷移 |
| フォロワー数 | `users.follower_count` |
| フォロー中数 | `users.following_count` |
| フォローボタン | フォロー状態に応じて「フォローする」「フォロー中」を切り替え。自分自身には表示しない |

### 3.3 検索結果が 0 件の場合

「該当するユーザーが見つかりません」のメッセージを表示する。

### 3.4 検索ワードが空の場合

「検索ワードを入力してください」のメッセージを表示し、API 呼び出しは行わない。

---

## 4. 使用 API エンドポイント

| Method | エンドポイント | パラメータ | 説明 |
|--------|--------------|-----------|------|
| GET | `/api/users/search` | `q=xxx` | ユーザー名で部分一致検索 |

### レスポンス例

```json
[
  {
    "id": 3,
    "username": "alice",
    "avatarUrl": null,
    "followerCount": 42,
    "followingCount": 18,
    "followedByCurrentUser": false
  }
]
```

---

## 5. バリデーション仕様

### フロントエンド

| 対象 | ルール | 動作 |
|------|--------|------|
| 検索ワード | 1 文字以上でないと検索ボタンを活性化しない | ボタン無効化 |

### バックエンド

| 対象 | ルール | HTTP ステータス |
|------|--------|---------------|
| 認証 | JWT が有効であること | 401 Unauthorized |
| クエリパラメータ `q` | 必須（空文字不可） | 400 Bad Request |

---

## 6. 制約・前提条件

| 分類 | 内容 |
|------|------|
| 認証 | ログイン済みユーザーのみ検索可能 |
| 検索対象 | ユーザー名（@username）のみ。メールアドレスは検索対象外 |
| 検索方式 | 部分一致（SQL の `LIKE '%xxx%'`） |
| 表示件数 | 上限を設けるか否かは実装時に決定 |

---

## 7. 対象外仕様

| 対象外機能 | 理由 |
|-----------|------|
| メールアドレスによる検索 | プライバシー配慮 |
| 投稿内容・コメントの検索 | スコープ外 |
| 検索履歴の保存 | スコープ外 |
| オートコンプリート / サジェスト | スコープ外 |
