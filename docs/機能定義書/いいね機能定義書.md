# いいね機能定義書

**バージョン:** 1.0
**作成日:** 2026-05-17
**作成者:** Nakata Saki

---

## 1. 機能概要

ログイン済みユーザーが任意の投稿に「いいね」をつけ・取り消しできる機能。
1 ユーザーにつき 1 投稿に対して 1 回のみいいね可能。
いいね数は投稿カードおよび投稿詳細画面に常時表示する。

---

## 2. ユーザーストーリー

| ID | ストーリー |
|----|---------|
| US-LIKE-01 | ログイン済みユーザーとして、投稿カードの「いいね」ボタンを押すことで、その投稿にいいねしたい |
| US-LIKE-02 | ログイン済みユーザーとして、いいね済みの投稿の「いいね」ボタンを再度押すことで、いいねを取り消したい |
| US-LIKE-03 | ログイン済みユーザーとして、各投稿に表示されるいいね数を見て、投稿の人気を確認したい |
| US-LIKE-04 | ログイン済みユーザーとして、自分がいいね済みの投稿のボタンが色付きになっていることで、いいね状態を判断したい |

---

## 3. 機能詳細

### 3.1 いいね操作

| 項目 | 内容 |
|------|------|
| 操作 | 投稿カードの「いいね」ボタンをクリック |
| 状態トグル | いいね済み → 取り消し、未いいね → いいね |
| 表示場所 | タイムライン（S-03）の投稿カード、投稿詳細画面（S-04） |

### 3.2 ボタン表示仕様

| 状態 | 表示 |
|------|------|
| 未いいね | ハートアイコン（線のみ） + いいね数 |
| いいね済み | ハートアイコン（塗りつぶし・強調色） + いいね数 |

### 3.3 いいね数の表示

| 項目 | 内容 |
|------|------|
| 表示場所 | 全ての投稿カード・投稿詳細画面 |
| データ源 | `posts.like_count`（非正規化カウンタ） |
| 更新タイミング | いいね/取り消し操作後にリアルタイム反映（楽観的更新） |

### 3.4 制限

| 項目 | 内容 |
|------|------|
| 1 ユーザー 1 いいね | 同じ投稿に複数回いいねすることはできない（DB 側で UNIQUE 制約） |
| 自分の投稿へのいいね | 可能 |

---

## 4. 使用 API エンドポイント

| Method | エンドポイント | 説明 |
|--------|--------------|------|
| POST | `/api/posts/{id}/likes` | いいねする |
| DELETE | `/api/posts/{id}/likes` | いいね取り消し |

### レスポンス例

```json
{
  "postId": 42,
  "likeCount": 13,
  "likedByCurrentUser": true
}
```

---

## 5. バリデーション仕様

| 対象 | ルール | HTTP ステータス |
|------|--------|---------------|
| 認証 | JWT が有効であること | 401 Unauthorized |
| 重複いいね | すでにいいね済みの場合は 200 を返す（エラーにしない） | 200 OK |
| 存在確認 | 対象投稿が存在すること | 404 Not Found |

---

## 6. 制約・前提条件

| 分類 | 内容 |
|------|------|
| 認証 | ログイン済みユーザーのみいいね可能 |
| 一意性 | `likes` テーブルの UNIQUE(post_id, user_id) で DB レベルで強制 |
| カウンタ更新 | `posts.like_count` はサービス層で `+1/-1` 更新する |

---

## 7. 対象外仕様

| 対象外機能 | 理由 |
|-----------|------|
| いいねしたユーザー一覧の表示 | スコープ外 |
| コメントへのいいね | スコープ外（投稿へのいいねのみ） |
