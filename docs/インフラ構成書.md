## 1. インフラ概要

本番環境は AWS 上に構築する。学習目的のため AWS 無料枠の範囲内での構築を基本方針とする。
画像ストレージには AWS S3 を使用し、ALB（Application Load Balancer）を EC2 前段に配置する。

> AWS 上への実際のデプロイは未定。本ドキュメントは構成の設計方針を定めるものとする。

## 2. アーキテクチャ図

```
インターネット
     │ HTTP (port 80)
     ▼
┌──────────────────────────────────────────┐
│  ALB（Application Load Balancer）         │
│  （パブリックサブネット・マルチ AZ）        │
└──────────────────────────────────────────┘
     │ HTTP (port 80)
     ▼
┌──────────────────────────────────────────┐
│  EC2（パブリックサブネット）               │
│                                          │
│  nginx（port 80）                        │
│  ├── React 静的ファイル配信              │
│  └── /api/* → Spring Boot（port 8080） │
│                     ↓                   │
│  Spring Boot（port 8080）               │
│  （systemd でプロセス管理）              │
└──────────────────────────────────────────┘
     │ PostgreSQL（port 5432）  │ HTTPS（S3 API）
     │ プライベートサブネット内のみ│ IAM ロール経由
     ▼                          ▼
┌─────────────────┐    ┌────────────────────┐
│  RDS            │    │  S3                │
│  PostgreSQL 16  │    │  画像ストレージ      │
│  （プライベート） │    │  （投稿画像）        │
└─────────────────┘    └────────────────────┘
```

## 3. AWS リソース構成

### 3.1 ネットワーク

| リソース | 説明 |
|---------|------|
| VPC | 本番環境専用の仮想ネットワーク |
| インターネットゲートウェイ | VPC をインターネットに接続 |
| パブリックサブネット（2 AZ） | EC2・ALB を配置。外部からアクセス可能 |
| プライベートサブネット（2 AZ） | RDS を配置。外部から直接アクセス不可 |
| ルートテーブル | パブリックサブネットのトラフィックを IGW に向ける |

### 3.2 セキュリティグループ

| 名前 | インバウンド | 用途 |
|------|------------|------|
| ALB 用 SG | 80（インターネットから） | ALB への外部アクセス制御 |
| EC2 用 SG | 80（ALB SG から）、22（SSH） | EC2 へのアクセスを ALB に限定 |
| RDS 用 SG | 5432（EC2 SG から） | DB アクセスを EC2 に限定 |

### 3.3 ALB（Application Load Balancer）

| 項目 | 内容 |
|------|------|
| タイプ | Application Load Balancer（HTTP） |
| 配置 | パブリックサブネット（マルチ AZ） |
| ターゲットグループ | EC2（port 80） |
| ヘルスチェック | GET `/actuator/health`（Spring Boot） |

### 3.4 EC2

| 項目 | 内容 |
|------|------|
| インスタンスタイプ | t2.micro（AWS 無料枠） |
| OS | Amazon Linux 2023 |
| ストレージ | 20GB（gp2） |
| 配置 | パブリックサブネット |
| IAM ロール | S3 バケットへの読み書き権限を付与 |

### 3.5 RDS

| 項目 | 内容 |
|------|------|
| インスタンスクラス | db.t3.micro（AWS 無料枠） |
| エンジン | PostgreSQL 16 |
| ストレージ | 20GB |
| 配置 | プライベートサブネット（2 AZ） |
| 外部アクセス | 不可（publicly_accessible = false） |
| 自動バックアップ | 7 日間保持（無料枠内） |

### 3.6 S3

| 項目 | 内容 |
|------|------|
| 用途 | 投稿画像のストレージ |
| アクセス方法 | EC2 の IAM ロール経由（アクセスキー不使用） |
| パブリックアクセス | 画像 URL を公開 or 署名付き URL で配信（要検討） |
| 対応フォーマット | JPEG / PNG / GIF（詳細はバリデーション仕様書を参照） |

## 4. EC2 上のソフトウェア構成

| ソフトウェア | 役割 |
|------------|------|
| Java 25（OpenJDK） | Spring Boot アプリの実行環境 |
| Spring Boot 4.0.5 | バックエンド API サーバー（port 8080） |
| systemd | Spring Boot プロセスの常駐・自動再起動管理 |
| nginx | リバースプロキシ（port 80 → Spring Boot 8080）・React 静的ファイル配信 |

## 5. ローカル開発環境

本番 AWS 環境とは別に、Docker Compose を使用してローカル開発環境を構築する。

```
開発者のブラウザ
     │
     ├── localhost:5173 → Vite（React 開発サーバー）
     │
     └── localhost:8080 → Spring Boot（ローカル起動）
                              ↓
                         localhost:5432 → PostgreSQL（Docker）
```

| 項目 | 内容 |
|------|------|
| PostgreSQL | Docker Compose（postgres:16） |
| React | Vite 開発サーバー（port 5173） |
| Spring Boot | ローカル直接起動（port 8080） |
| 画像ストレージ | AWS S3（開発・本番共通）または LocalStack で代替 |

## 6. デプロイフロー（参考）

### 6.1 初回構築（予定）

1. AWS 上に VPC / ALB / EC2 / RDS / S3 を作成
2. EC2 に Java 25・nginx をインストール
3. Spring Boot JAR をビルドし EC2 へ転送
4. React をビルドし静的ファイルを EC2 の nginx 配信ディレクトリへ転送
5. systemd でSpring Boot を起動・自動起動設定
6. nginx を起動し reverse proxy 設定を反映

### 6.2 コード変更時のデプロイ（予定）

1. バックエンド: JAR をビルドし EC2 へ転送 → systemd restart
2. フロントエンド: `npm run build` → 静的ファイルを EC2 へ転送

## 7. セキュリティ方針

| 項目 | 対応 |
|------|------|
| DB パスワード | 環境変数で管理（`.env` は Git 管理外） |
| RDS アクセス | EC2 のセキュリティグループからのみ許可（インターネット非公開） |
| SSH アクセス | 公開鍵認証のみ（パスワード認証なし） |
| S3 アクセス | IAM ロールを使用（アクセスキーを EC2 に保持しない） |
| JWT シークレット | 環境変数で管理 |
| HTTPS | 学習目的のため未対応（HTTP のみ） |
