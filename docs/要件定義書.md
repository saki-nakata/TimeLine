## 1. プロジェクト概要

### 1.1 プロジェクト名

TimeLine

### 1.2 目的

X（旧Twitter）をモデルにした、テキストおよび画像を投稿できる SNS タイムラインアプリを作成する。
複数ユーザーが利用し、投稿・コメント・いいね・フォローなどのソーシャル機能を通じてユーザー同士が交流できる Web アプリケーションを提供する。

### 1.3 背景

プログラミングスクール中級編の課題として開発する。
React・Spring Boot・PostgreSQL・AWS を組み合わせ、フロントエンドからインフラまで一連の Web アプリケーション開発の流れを体験することを目的とする。

### 1.4 対象ユーザー

- スクール受講生
- 個人ユーザー
- スマートフォン・PC の両方から利用するユーザー

## 2. システム概要

### 2.1 システム構成

```
[ユーザー（ブラウザ）]
        ↕ HTTP
[ALB（Application Load Balancer）]
        ↕ HTTP
[EC2]
  ├── フロントエンド（React 静的ファイル / nginx 配信）
  └── バックエンド（Spring Boot API / port 8080）
        ↕ MyBatis
[PostgreSQL（RDS）]
        ↕ S3 API（IAM ロール）
[AWS S3（画像ストレージ）]
```

### 2.2 技術スタック

| 役割 | 技術・バージョン |
|------|----------------|
| フロントエンド | React 19.2.5 + TypeScript 6.0.3 |
| ビルドツール（FE） | Vite 8.0.10 |
| スタイリング | Tailwind CSS 3.4.19 |
| HTTP クライアント | Axios 1.7.9 |
| バックエンド | Spring Boot 4.0.5 |
| 言語 | Java 25 |
| ビルドツール（BE） | Gradle 9.4.1（Kotlin DSL） |
| O/R マッパー | MyBatis Spring Boot Starter 4.0.0 |
| 認証 | JWT（HttpOnly Cookie）※方式は後日確定 |
| データベース | PostgreSQL 16 |
| 画像ストレージ | AWS S3 |
| コンテナ（開発） | Docker Compose v2（PostgreSQL のみ） |
| ホスティング | AWS EC2 + RDS + ALB（デプロイするかは未定） |

※ AWS インフラの詳細は [インフラ構成書.md](インフラ構成書.md) を参照。

## 4. 非機能要件

| 分類 | 要件 |
|------|------|
| レスポンシブ対応 | スマートフォン・タブレット・PC のいずれでも正常動作すること |
| パフォーマンス | タイムライン初期表示は 3 秒以内 |
| セキュリティ | パスワードは bcrypt でハッシュ化して保存する |
| セキュリティ | 認証が必要な API は JWT 検証を行う |
| セキュリティ | アップロード画像の MIME タイプ・サイズを API 側で検証する |
| セキュリティ | SQL インジェクション対策は MyBatis のパラメータバインディングで担保する |
| 可用性 | AWS 無料枠（12ヶ月）の範囲内で動作すること |
| 開発環境 | Docker Compose 1 コマンドでローカル DB を起動できること |

## 7. 用語定義

| 用語 | 説明 |
|------|------|
| 投稿 | テキスト（最大 280 文字）と任意の画像 1 枚で構成される 1 件のデータ |
| タイムライン | 投稿を新着順に一覧表示する画面 |
| いいね | ユーザーが投稿に対して行う評価アクション。1 ユーザー 1 投稿につき 1 回のみ |
| コメント | 投稿に対して他のユーザーが追加するテキストの返信 |
| フォロー | 特定のユーザーの投稿をフォロー中タイムラインに表示するためのアクション |
| フォロワー | 自分をフォローしているユーザー |
| フォロー中 | 自分がフォローしているユーザー |
| App Router | React Router や Vite Router によるクライアントサイドルーティング |
| JWT | JSON Web Token。認証情報をトークン形式で管理する仕組み |
| HttpOnly Cookie | JavaScript からアクセスできない Cookie。XSS 対策として JWT の格納に使用 |
